## 一、选题策略

### 核心特点：从日常切入，上升到哲学

| 篇文章 | 日常切入点 | 深层主题 |
|--------|-----------|----------|
| 好软件 | 两年没装新软件 | 成品软件（finished software）的价值 |
| 戒手机 | 乡下的竹篮子 | 物理环境对行为的影响 |
| 基础设施 | 经济垃圾时间 | 基础设施投资是理性乐观 |
| 阅读清单 | 年度盘点 | 知识背景的重要性 |

**模式**：个人经验 → 抛出反直觉观点 → 理论论证 → 实践支撑

## 二、观点输出

### 鲜明，敢于下判断

- "年度榜单永远追新软件，但软件领域近年越来越无聊"
- "Adobe 是 PDF 问题的兜底解药，说到底，这些问题本就是 Adobe 制造的"
- "Microsoft Word，存了太多聊天记录而不敢更换的破手机，脏乱差的小区"
- "AI 再快，承担责任的还是你"

### 特点

1. **有对立面**：明确反对什么、支持什么
2. **有数据/案例支撑**：不是空对空
3. **有延伸思考**：从一个点联想到更多

## 三、知识密度

### 旁征博引，但为我所用

- **引用名人**：赖特·米尔斯、狄德罗、维特根斯坦、Paul Graham
- **引用论文/书籍**：《The Lab Book》、《逻辑哲学论》
- **引用英文原文**：大段引用英文文章作为论据
- **引用历史/商业案例**：宜家模块化货架、乔布斯的皮克斯收购

### 引用原则

- 引用不是为了炫耀，而是服务论点
- 每引用一个观点，必有个人解读
- 脚注详细，标注来源

## 四、语言风格

### 学术化但不晦涩

**用词特点**：
- 精准：喜欢用学术术语（模块化、可证伪性、基础设施）
- 偶尔冷幽默："真是太凄惨了！"、"足以让我破产甚至牢底坐穿"
- 律师思维：钻概念、讨论定义

**句式特点**：
- 长句居多，复杂但清晰
- 喜欢用"……不过……"、"……而……"、"……不过，并非……"这种转折句式
- 段落内部往往先抑后扬

## 五、结构特点

### 层级清晰的章节式写作

```
## 大章节标题
### 二级小节
#### 可选三级

- 列表项
- 列表项

正文内容...
```

### 固定结构

1. **开篇**：从个人经历/新闻切入，抛出核心观点
2. **正文**：分多个维度展开，每个维度有理论+案例
3. **小结**：段落内小结
4. **结语**：总结 + 升华 + 开放式结尾

### 列表使用

- 少用无序列表，更多用自然段落叙述
- 列表主要用于列举工具、案例、参考链接

## 六、排版偏好

### 标题体系

- `#` 一级：文章总标题
- `##` 二级：大章节
- `###` 三级：小节
- `####` 四级：偶尔使用

### 链接处理

- 段落内嵌链接：`[文本](链接)`
- 专用"拓展阅读"段落：罗列多篇相关文章链接

### 脚注系统

- 使用 `[^数字]` 格式
- 文末统一返回注释
- 脚注内容丰富，不只是出处说明，还有延伸评论

### 引用块

- 英文原文使用引用块突出
- 保留原文语言（英文为主）

## 七、配图偏好

### 图片类型

| 类型 | 示例 | 风格 |
|------|------|------|
| 实拍图 | 竹篮子、厨房货架、白板墙纸 | 生活化、真实 |
| 截图 | 软件界面、网页 | 功能说明 |
| 对比图 | 改造前后 | 直观展示 |

### 图片说明

- 详细且有上下文
- 会说明为什么放这张图
- 例："由于我不可能用手机拍摄篮子中的该手机，所以图中乃家人的手机代为出镜"

### 配图原则

- 每张图都有明确目的
- 不追求精美，但追求真实
- 自拍 > 网图

## 八、学习要点

### 可以借鉴

1. **从日常切入**：从小事引出大道理，读者有代入感
2. **观点鲜明**：敢于下判断，有对立面
3. **知识引用**：但服务论点，不炫技
4. **结构清晰**：章节式，阅读友好
5. **脚注丰富**：增加可信度，展示阅读量
6. **配图真实**：用实拍图，有生活气息

### 需要注意

1. **不适合所有人**：这种风格需要知识储备和思考深度
2. **阅读门槛较高**：不适合碎片化阅读
3. **更新频率低**：高质量长文需要时间打磨

## 十一、写作套路总结

### 万能模式：个人经历 → 抛出问题 → 分析问题 → 给出方案 → 升华主题

1. **开头**：从一件具体的小事切入（自己的经历、遇到的问题）
2. **问题**：这个问题大多数人也会遇到，但解决方式往往是错的
3. **分析**：从理论/原理/多个角度分析为什么
4. **方案**：给出具体的、可操作的解决方案
5. **升华**：上升到方法论或人生态度

### 文章长度

- 工具教程类：1500-3000 字
- 方法论/思维类：3000-5000 字
- 年度盘点类：4000-6000 字
- 特点：文章都不短，但读起来不累，因为段落短、信息密度高

## 十二、进阶学习要点

### 还需要提升

1. **概念深度**：Minja 的文章常常讨论一个概念的定义和边界
2. **多角度论证**：同一个问题，从技术、历史、商业、个人体验多个角度分析
3. **专业背景融入**：律师身份对事物的分析方式（证据、论证、逻辑）
4. **长期实践积累**：很多方法都是多年实践总结出来的
